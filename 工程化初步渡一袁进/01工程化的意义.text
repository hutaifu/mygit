# 工程化意义
+   商业模式（盈利模式） =>具体形态，产品=》实现=》交付

+   实现环节，设计到很多部门岗位，测试，ui,运维，  涉及到组织管理能力=》发挥资源的最大效能
+   效能的方向指向交付，效能需要发挥出来，不能摸鱼，不能干成本高，回报低的诗，同时方向不能混乱，
否则体现的就是碌碌无为，相互推责，拉帮结派，扯皮打架，非常考验管理者的管理水平，
+   管理会用到管理工具，sop标准化流程,打孔器，都是降低管理成本，提升效能
+   公司规章制度，部门的规章制度，代码开发规范，组员和组员之间的协作方式都和管理相关
+   技术管理层面，使用对应工具来处理不同的开发问题，处理兼容性，开发规范，命名规范等问题就是前端
工程化。
+   前端工程化=》前端开发的管理工具，项目规模越大，团队成员越多，工程化收益越大。
